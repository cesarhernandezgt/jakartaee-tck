Introduction

This plan documents the high level Compatibility Test Plan for the Java API for XML Registries(JAXR) Version 1.0 Specification which defines APIs for creating, publishing and querying XML Registries. Both the Technology Compatibility Kit (TCK) and the Compatibility Test Suite (TS) comprise a suite of tests in which the test cases are derived from the requirements of the JAXR specification. These TCK and TS tests are written to ensure application portability and compatibility among JAXR implementations based on this API. The reference implementation will validate that the specifications can be implemented, this will provide a reference point for the development community, and will be used for developing Compatibility Tests, which leads to a brand. This ensures Sun's, specifically Java's WORA (Write Once Run Anywhere) mission.

Goals

Provide compatibility tests (the Technology Compatibility Kit - TCK) and (the Compatibility Test Suite - TS) for the set of functionality as defined in the JAXR Specification Version 1.0. These tests will verify the JAXR API's.

Non-Goals

  •     Performance
  •     Scalability
  •     Ease of use
  •     Non-required (implementation specific) areas of the specification
  •     Optional features.


Types of Compatibility Tests

There are 3 types of tests that make up the Compatibility Test Suite, these include:  
  • Signature Tests - Checks that all of the required API signatures are available for an implementation, no supersets or subsets

  • API Tests - Verifies that all of the required classes and methods behave according to the specification on a method by method basis.

  • End-To-End Tests - A new type of compatibility test that was introduced with the J2EE platform. These type of tests span across N-tiers of the J2EE platform.

    For the Technology Compatibility Kit (TCK), we will be doing Signature, API, End-To-End tests for JAXR 1.0 technology release.

    For the Compatibility Test Suite (TS), we will also be doing Signature, API, End-To-End tests for the JAXR 1.0 technology release integrated within the J2EE 1.4 platform. The TS tests will be run in ejb/web and appclient vehicles. Additionally, there will be ebXML (may be optional??) added for the J2EE 1.4 platform.



Compatibility Test Suite Releases

There will be 2 types of Compatibility Test Suite releases for JAXR, they are:  
  • JAXR 1.0 TCK - Technology Compatibility Kit just for JAXR 1.0 technology release

  • J2EE 1.4 TS - Compatability Test Suite for J2EE 1.4 Platform which will contain the tests for the JAXR 1.0 technology integrated within the J2EE 1.4 Platform.

    For the JAXR 1.0 TCK we will be using the Registry Server for a reference UDDI V2 server.

    For the J2EE 1.4 TS we will be using the J2EE 1.4 Platform for testing. End-To-End testing and Interoperability testing will be the main focus of the J2EE 1.4 TS. The Signature and API tests from the JAXR 1.0 TCK will also be included in the J2EE 1.4 TS.

High Level Test Areas

The following features of JAXR will be tested as specified:

Features Tested for TCK in JAXR 1.0

  • Infomodel Classes
  • Classification
  • Association
  • Connection
  • LifeCycle
  • Query

Features Tested for TS in J2EE 1.4

  • All of the above
  • J2EE Container(s) Focus (EJB/WEB) for Compatibility Tests
  • Additional End-To-End Testing

Features Not Tested

  • Implementation specific and optional features.

Test Approach

The test approach will utilize the JAXR 1.0 Technology Release  to develop the JAXR tests for both the JAXR TCK and the J2EE 1.4 TS. The JAXR tests require a Registry for execution. As stated above in the section Compatibility Test Suite Releases we will use the JWSDP Registry Server for the JAXR TCK and the J2EE 1.4 Platform environment for the J2EE 1.4 TS.
